# mdeasy 技术方案（完整版）

> **产品**：本地优先的 Markdown **阅读器**（不是编辑器 / 笔记库 / 第二大脑）  
> **对标**：[MDView](https://www.mdview.cn/) — 小体积、快启动、不登录、不联网、不打扰、专注阅读  
> **平台**：仅 **macOS**  
> **架构**：方案 A — **Swift 薄壳 + WKWebView + 静态 reader**  
> **工程**：本地无 Xcode 亦可开发；**GitHub Actions** 编译打包  
> **分发（当前）**：**无 Apple Developer 证书**；自用 unsigned 构建 + **系统「仍要打开」**

---

## 0. 决策一览（已拍板）

| 议题 | 决策 |
|------|------|
| 产品形态 | 阅读器 only，打开 `.md` 即读 |
| 平台 | 仅 macOS（不做 Win / Android / iOS） |
| 体积目标 | 基础包约 **1.5–3 MB**（对标 MDView ~1.7MB 量级） |
| 壳 | **Swift + AppKit + WKWebView**（非 Tauri / Electron） |
| 前端 | **静态 HTML/JS**（esbuild），不用 Svelte/React |
| Mermaid | **支持，但不进基础包**（完整包或本地引擎导入） |
| 网络 | 默认零请求；禁止 CDN；禁止首次联网下引擎 |
| 本机环境 | **无 Xcode / 无 Swift 工具链也可**，以编辑 + git 为主 |
| 编译打包 | **GitHub Actions `macos-latest`** |
| 证书 / 年费 | **当前不要**；不做 Developer ID 签名与公证 |
| 自用打开方式 | **方式 B**：系统设置 → 隐私与安全性 → **仍要打开** |
| 商业 | 自用 / 免费工具向；无账号、无遥测 |

## 1. 产品目标与边界

### 1.1 要做什么

| 目标 | 具体含义 |
|------|----------|
| 小体积 | 基础 `.app` / `.dmg` 约 **1.5–3 MB** |
| 快启动 | 冷启动到可读 **< 1s**（SSD、中等文档） |
| 不登录 | 无账号、无云同步 |
| 不联网 | 零业务网络请求；资源全部本地 |
| 不打扰 | 无广告、无遥测、无强制更新、无营销弹窗 |
| 专注阅读 | 大纲、主题、滚动记忆；不提供编辑 |
| 实时预览 | 外部编辑器保存 → 自动重渲染 |
| Mermaid | 流程图等，**拆包**提供（见 §6） |
| 多主题 | 亮 / 暗 / 纸质 / 护眼（纯 CSS） |
| HTML 导出 | 单文件自包含 HTML |

### 1.2 明确不做

- Windows / Android / 多端抽象  
- WYSIWYG 编辑、知识库、双向链接、插件市场、AI  
- Apple Developer 签名公证（**当前阶段**；公开分发时再议）  
- 依赖本机安装 Xcode 才能参与开发  
- 基础包内置完整 mermaid / KaTeX / 全语言高亮  

### 1.3 与 MDView 的关系

| 维度 | MDView | mdeasy |
|------|--------|--------|
| 平台 | Win + Android | **仅 macOS** |
| 体量 | ~1.7 MB | 目标同一量级 |
| 结构 | 系统 WebView + 薄壳 | **同构**（Swift + WKWebView） |
| 分发 | 安装包；Win 有 SmartScreen 问题 | 自用 unsigned +「仍要打开」 |

## 2. 总体架构（方案 A）

### 2.1 结构

```
┌────────────────────────────────────────────────────────────┐
│  mdeasy.app                                                │
│  Swift / AppKit                                            │
│  · NSWindow + WKWebView                                    │
│  · 文件关联 · 菜单 · 拖放 · 单实例                           │
│  · 读文件 · 编码 · baseDir 沙箱 · FSEvents 监听 · 导出写盘   │
│  · WKScriptMessageHandler 桥接                             │
└────────────────────────────▲───────────────────────────────┘
                             │ JS bridge
┌────────────────────────────┴───────────────────────────────┐
│  Resources/reader/   （纯静态，无 Node 运行时）               │
│  index.html · app.js · md.js · hljs-lite.js · themes.css   │
│  （mermaid 不进基础包）                                      │
└────────────────────────────────────────────────────────────┘
```

**原则**：系统 WebView 负责渲染；原生壳只做窗口 / 文件 / 安全；前端是静态阅读页，不是重型 SPA。

### 2.2 为何不用 Tauri / Electron

| 方案 | 体积 | 无本机 Xcode | 自用 unsigned | 结论 |
|------|------|--------------|---------------|------|
| **Swift + WKWebView** | **最优 ~1.5–3MB** | CI 编译即可 | 可以 | **采用** |
| 极简 Tauri | 多 1–2MB+ | 本机也可用 Rust，但 Mac 链仍重 | 可以 | 备选，已否 |
| Electron | 80MB+ | 可 | 可以 | 否 |

仅 Mac + 极致体积 → **Swift 薄壳** 最合适。

### 2.3 核心用户流程

```
用户双击 xxx.md
  → Launch Services 启动 mdeasy，传入 path
  → Swift 校验可读，读入文本，探测编码 → UTF-8
  → WKWebView 已加载本地 reader/index.html
  → bridge 下发 { path, text, baseDir }
  → JS：markdown-it 渲染 → 大纲 → 主题样式
  → Swift FSEvents 监听 path；变更则再次下发 text
  → 用户：切换主题 / 大纲跳转 / 导出 HTML / 用编辑器打开
```

## 3. 功能规格

### P0 — 基础包（~1.5–3 MB）

- 打开 `.md` / `.markdown`（可选 `.mdx` 当纯文本 md）  
- CommonMark + GFM（表格、任务列表、删除线、自动链接）  
- 本地相对路径图片（限制在 md 所在目录树）  
- 代码高亮（语言子集，或 P0 极简 CSS）  
- 浮动大纲 H1–H3 + 滚动高亮  
- 亮 / 暗主题 + 记忆  
- 文件变更实时刷新  
- 拖放打开、Dock 打开、文件关联  
- HTML 导出（无 mermaid 时图块保留源码）  
- 完全离线、无遥测  

### P1

- Mermaid（完整包或 Application Support 引擎文件）  
- 纸质 / 护眼主题、字号、行宽、滚动位置记忆  
- 在 Finder 中显示、用默认编辑器打开  
- 最近文件列表  

### P2

- 多窗口、目录树只读浏览  
- YAML front matter 展示  
- 系统打印 → PDF  
- 自定义 CSS、KaTeX 外置引擎

## 4. 技术栈

### 4.1 原生壳（Swift）

| 项 | 选择 |
|----|------|
| 语言 | Swift 5.9+ |
| UI | **AppKit**（菜单 / 文件关联更直接） |
| 容器 | **WKWebView** |
| 桥接 | `WKScriptMessageHandler` + `evaluateJavaScript` |
| 文件监听 | FSEvents / `DispatchSource`，debounce 150–300ms |
| 编码 | 优先 UTF-8；必要时简单探测 |
| 最低系统 | **macOS 12+** |

### 4.2 静态 reader（JS）

| 能力 | 选择 | 体积策略 |
|------|------|----------|
| 构建 | **esbuild** minify | 1–2 个 bundle |
| Markdown | **markdown-it** + 少量插件 | 单文件打包 |
| 高亮 | highlight.js **语言白名单** | 只留常用 8–12 种 |
| 主题 | CSS 变量 | < 10 KB |
| Mermaid | **不进基础包** | 见 §6 |
| 框架 | **无** React/Svelte | 状态极少，手写即可 |

### 4.3 安全

- 默认不渲染 markdown 原始 HTML（或消毒）  
- 图片 URL 仅映射 **baseDir** 下文件，禁止 `../` 逃逸  
- CSP 示例：

```
default-src 'none';
script-src app:;
style-src app: 'unsafe-inline';
img-src app: data: blob:;
font-src app: data:;
connect-src 'none';
frame-src 'none';
base-uri 'none';
```

（`app:` 为自定义 URL scheme 或 `WKURLSchemeHandler`，实现时以实际 scheme 为准。）

- 应用业务代码不发起网络请求；不引入更新检查  

### 4.4 桥接 API 草案

```ts
// Swift → JS（事件）
{ type: 'doc', path, baseDir, text, encoding, mtimeMs }
{ type: 'file-changed', path, text, mtimeMs }
{ type: 'theme', name }

// JS → Swift（message handler 名如 "mdeasy"）
{ type: 'ready' }
{ type: 'export-html', html, suggestedName }
{ type: 'open-in-editor', path }
{ type: 'reveal-in-finder', path }
{ type: 'set-preference', key, value }
{ type: 'error', message }
```

Swift 侧 commands 等价能力：打开文件对话框、写导出文件、改偏好（存 Application Support）。

## 5. Mermaid 拆包策略

| 策略 | 基础包体积 | 是否联网 | 采用 |
|------|------------|----------|------|
| 基础包不含；另提供 **full dmg** | 最小 | 否 | **主推** |
| 用户从 Release 下载 `mermaid.min.js`，导入到 App Support | 最小 | 否 | 可并行 |
| 首次运行自动下载 | 最小 | **是** | **禁止** |

行为：

- 文档含 \`\`\`mermaid 且引擎可用 → 渲染  
- 引擎不可用 → 显示源码块 + 一行说明「当前为基础版 / 未加载图表引擎」（不弹窗骚扰）

## 6. 体积预算

### 6.1 基础包拆解目标

| 部分 | 预算 |
|------|------|
| Swift 可执行文件（Release + strip） | 0.4–1.2 MB |
| reader 静态资源（无 mermaid） | 150–350 KB |
| 图标 / plist / 杂项 | < 150 KB |
| **.app** | **~1.0–2.0 MB** |
| **.dmg（压缩）** | **常 ~0.8–1.8 MB** |

Full 包 ≈ 基础 + mermaid **+1–2 MB**。

### 6.2 性能预算

| 指标 | 目标 |
|------|------|
| 冷启动窗口可见 | < 300–400 ms |
| 50KB md 可读 | < 800 ms–1 s |
| 保存后预览更新 | < 300 ms（含 debounce） |

### 6.3 CI 体积门禁

- 基础配置构建：`.app` 超过 **3 MB** → job 失败  
- 在 Actions 日志打印 `du -sh`、主要文件体积

## 9. GitHub Actions 打包

### 9.1 目标产物（无证书）

| 产物 | 说明 |
|------|------|
| `mdeasy.app` | Release 配置，**unsigned / ad-hoc** |
| `mdeasy-x.y.z.dmg` | 仅打包容器，**不签名、不公证** |
| 构建日志中的体积信息 | 门禁与回归 |

### 9.2 `ci.yml`（PR / push）

职责：

1. `actions/checkout`  
2. `setup-node` → `reader` 下 `npm ci && npm run build`  
3. 同步 `reader/dist` → `App/Resources/reader`  
4. 若使用 XcodeGen：`xcodegen generate`  
5. `xcodebuild -scheme mdeasy -configuration Release -derivedDataPath build ...`  
6. 组装 `.app`，`du -sh`，体积门禁  
7. `actions/upload-artifact` 上传 `.app`（可选 zip）  

**不做**：import 证书、`codesign` Developer ID、`notarytool`。

### 9.3 `release.yml`（push tag `v*`）

1. 同 ci 构建  
2. `scripts/package-dmg.sh` 生成 dmg（`hdiutil`）  
3. `softprops/action-gh-release` 上传  
   - `mdeasy-x.y.z.dmg`  
   - 可选 `mdeasy-x.y.z-full.dmg`  
4. Release 正文自动带上 **首次打开说明（方式 B）** 模板  

### 9.4 dmg 打包（无签名）

```bash
# 概念步骤（scripts/package-dmg.sh）
# 1. 将 mdeasy.app 放入临时文件夹（可附 Applications 软链）
# 2. hdiutil create -volname mdeasy -srcfolder ... -ov -format UDZO mdeasy-x.y.z.dmg
```

dmg **只是分发容器**，不引入证书要求。

### 9.5 Runner 与成本

- `runs-on: macos-latest`（自带 Xcode）  
- 公开仓库 macOS 分钟额度以 GitHub 现行政策为准；私有库注意计费  
- PR 只编 app、不打 dmg/公证，缩短反馈时间  

### 9.6 工程冷启动（完全无 Xcode 时）

任选：

1. 仓库内提交 **最小 WKWebView 模板工程**（推荐，一次到位）  
2. **XcodeGen** 的 `project.yml` 在 CI 生成 `xcodeproj`  
3. 借 Mac / 云 Mac **一次** 用 Xcode 建壳后永不本地打开  

之后你只需改 `Sources/*.swift` 与 `reader/`。

## 10. 自用分发与打开方式（方式 B）

### 10.1 当前分发策略

```
GitHub Actions 构建 unsigned .app / .dmg
  → 上传 Artifact 或 GitHub Release
  → 你在自己的 Mac 下载
  → 首次用「方式 B」信任
  → 日常双击使用
```

**不需要** Apple Developer 年费、证书、公证。

### 10.2 方式 B：系统设置 → 仍要打开（标准流程）

适用于：下载后双击出现「无法验证开发者」/ 被拦截。

1. 双击 `mdeasy.app`（或 dmg 里的 app）→ 系统拦截，点「完成」/「取消」即可。  
2. 打开 **系统设置（System Settings）**  
3. 进入 **隐私与安全性（Privacy & Security）**  
4. 向下滚动到安全性相关区域  
5. 应看到类似：  
   **“mdeasy”已拦截，因为来自身份不明的开发者**  
6. 点击 **「仍要打开」（Open Anyway）**  
7. 在确认框中再次选择 **打开**  
8. 之后在同一台 Mac 上一般可直接打开  

> 说明：macOS 文案随版本略有差异（Ventura / Sonoma / Sequoia），位置都在 **隐私与安全性** 页下半部。

### 10.3 方式 B 的适用范围与限制

| 项 | 说明 |
|----|------|
| 是否要年费 | **不要** |
| 谁适合 | **自己用**；少数熟人可照说明操作 |
| 换机 / 重装 | 新机器需再走一次「仍要打开」 |
| 公司 MDM 管控设备 | 可能禁止未签名 app，需自管 |
| 公开给海量用户 | **不合适**；应以后上 Developer ID + 公证 |

### 10.4 明确不采用的打开方式（文档可不主推）

| 方式 | 原因 |
|------|------|
| A. `xattr -cr` 去隔离属性 | 有效，但偏命令行；**本方案文档主推方式 B** |
| C. 右键 → 打开 | 亦可，作为补充写在 README 即可 |
| 关闭系统完整性 / 乱关 SIP | **禁止**，不安全 |

### 10.5 README / Release 固定说明文案（建议原文收录）

```markdown
## 首次打开（未签名自用版）

本构建未使用 Apple 开发者证书签名，仅供自行使用。

1. 打开 mdeasy（若提示无法验证开发者，先关闭提示）
2. 打开「系统设置」→「隐私与安全性」
3. 在页面下方找到 mdeasy 的拦截提示
4. 点击「仍要打开」，并在对话框中确认

同一台 Mac 之后通常可直接双击启动。
```

### 10.6 与「正式签名版」的边界

| | 当前（自用） | 未来可选（公开） |
|--|--------------|------------------|
| 证书 | 无 | Developer ID Application |
| 公证 | 无 | notarytool + stapler |
| 用户操作 | 方式 B 一次 | 双击即可 |
| CI | 只 build + dmg | 增加 keychain import / codesign / notarize |
| 年费 | 0 | Apple Developer Program |

当前技术方案 **不实现** 签名链路；预留 `release.yml` 扩展点即可，避免过早复杂化。

## 13. 实施路线图

### Phase 0 — 骨架（2–3 天）

### Phase 3 — 可选

## 14. 风险与对策

| 风险 | 对策 |
|------|------|
| 无本机 Xcode，Swift 迭代慢 | 壳保持极薄；逻辑尽量放 reader；CI 快速反馈 |
| 未签名劝退他人 | 定位自用；README 写清方式 B；不对外假装「正式版」 |
| 系统升级后拦截文案/入口变化 | 大版本系统后更新 README 截图式说明 |
| Mermaid 拆包被觉残缺 | 提供 full 产物；应用内一行说明即可 |
| pbxproj 难维护 | XcodeGen 或固定模板少改工程结构 |
| 体积回涨 | CI 门禁；mermaid/hljs 严禁无脑全量 |

## 15. 成功指标

- 基础 dmg/app 进入 **~1.5–3 MB**  
- 冷启动 **< 1s** 可读  
- 断网 100% 可用  
- 无 Xcode 可开发 reader；push 即可获得 Mac 构建  
- **无年费** 下自己稳定使用（方式 B 仅首次）  
- 单人可维护：依赖少、无后端、无多端

## 16. 决策摘要

> **mdeasy = macOS 上的 MDView 哲学实践：Swift 极薄壳 + 系统 WKWebView + 静态 JS 渲染；Mermaid 拆包保体积；本地可不装 Xcode，用 GitHub Actions 产出 unsigned dmg/app；无 Apple 证书，自用通过「系统设置 → 隐私与安全性 → 仍要打开」完成首次信任。坚决不做第二 Obsidian，也不做当前阶段的签名公证复杂度。**
